## 面向 3D 存算一体硬件的智能编译映射与调度框架

---

## 摘要

本报告针对 3D 存算一体（CIM）硬件，设计并实现了一套智能编译器/调度器框架。该框架能够将深度学习模型（特别是 MoE 大模型）的权重高效映射到分层 3D 异构资源空间，并优化端到端推理延迟。框架包含模型解析、权重切分、MoE 激活轨迹分析、3D 映射算法、流水线调度和硬件模拟等核心模块。实验结果表明，该框架能够成功处理简单模型和 MoE 模型，并生成满足所有物理约束的映射与调度方案。

## 1. 引言

### 1.1 研究背景与意义

随着 Llama 70B、Mixture-of-Experts (MoE) 等大语言模型的爆发，模型参数量已突破千亿级别，传统的计算架构面临着严峻的"存储墙"与"功耗墙"挑战。

存算一体 (Computing-in-Memory, CIM) 技术通过将计算融入存储介质（如 3D NAND Flash），实现了"位置即算力"的革新。然而，这种架构引入了一个全新的数学挑战：如何在极度受限的 3D 物理空间中，高效地排布和调度巨大的神经网络模型？

### 1.2 研究目标

本赛题要求设计一个智能编译器/调度器，将复杂的深度学习模型（计算图）映射到一个抽象的分层 3D 异构资源空间中。这不仅是一个内存分配问题，更是一个涉及并行计算、流水线优化和异构调度的系统级难题。

### 1.3 典型场景：DeepSeek-671B

DeepSeek-V3 拥有 671B 总参数，采用 Fine-Grained MoE 架构（256 个路由专家，每次激活 8 个路由专家和一个共享专家）。这是典型的 3D CIM 部署挑战场景：

| 特性 | 挑战 |
|------|------|
| Expert 数量极多但单个 Expert 较小 | 需要精细的空间划分策略 |
| Top-K 路由机制 | 专家激活呈现高度稀疏性 |
| 互斥性与共现性 | 某些专家极少同时被选中 |
| 共享专家 | 始终激活的"热点" |

## 2. 工作原理与数学建模

### 2.1 硬件模型：3D 资源空间

我们将硬件抽象为一个三级层级的 3D 阵列结构：

#### 2.1.1 全局空间 (Global Space)

整个硬件资源被划分为 N×N 个相互独立的"存算核"(Sub-Cubes)，所有 Sub-Cubes 可以同时工作。

#### 2.1.2 存算核 (Sub-Cube)

每个 Sub-Cube 内部可以放置多个"权重立方体"(Weight-Cube)，在任意时刻只能激活唯一的一个 Weight-Cube。

#### 2.1.3 权重立方体 (Weight-Cube)

Weight-Cube 是神经网络权重矩阵（或其切分后的部分）在 3D 空间中的映射单元。

### 2.2 数学建模

#### 2.2.1 问题定义

给定：
- 模型计算图 $G = (V, E)$，其中 $V$ 为算子集合，$E$ 为依赖关系
- 3D 资源空间 $S = \{SC_0, SC_1, ..., SC_{N^2-1}\}$，每个 $SC_i$ 具有容量 $H \times W \times D$
- 每个算子 $v \in V$ 对应的权重矩阵 $W_v$

求解：
- 映射函数 $M: V \rightarrow S \times \mathbb{Z}^3$，将每个算子的权重映射到具体的 Sub-Cube 和 3D 坐标
- 调度函数 $T: V \rightarrow \mathbb{Z}$，确定每个算子的执行时间

#### 2.2.2 约束条件

1. **Weight Stationary**：权重一旦映射，不可动态迁移
2. **One Weight-Cube Per Sub-Cube**：$\forall t, \forall s \in SubCubes: \|Active(s, t)\| \leq 1$
3. **Strict Dependency**：下游算子必须等待上游所有 Section 输出就绪

#### 2.2.3 优化目标

$$\min \sum_{v \in V} (T_{end}(v) + \sum_{switches} D)$$

其中 $D$ 为 Sub-Cube 内切换开销。

### 2.3 MoE 激活轨迹分析

#### 2.3.1 共现矩阵

$$C_{ij} = \sum_{k=1}^{N} \mathbb{I}(E_i \in Trace_k \land E_j \in Trace_k)$$

其中 $E_i, E_j$ 为 Expert，$Trace_k$ 为第 $k$ 次推理的激活轨迹。

#### 2.3.2 激活频率

$$F_i = \sum_{k=1}^{N} \mathbb{I}(E_i \in Trace_k)$$

### 4.2 权重切分

#### 4.2.1 切分策略

当权重矩阵超出单个 Sub-Cube 的 2D 容量时：

- **行切分**：沿输入特征维度切分
- **列切分**：沿输出特征维度切分

#### 4.2.2 深度计算

$$Depth = \lceil \frac{Rows \times Cols}{H \times W} \rceil$$

#### 4.2.3 关键代码实现

```python
class WeightPartitioner:
    """权重切分器"""
    def partition(self, operator: Operator, config: HardwareConfig) -> List[WeightSection]:
        max_rows = config.subcube_hw
        max_cols = config.subcube_hw
        
        rows, cols = operator.weight_shape
        sections = []
        
        # 计算需要切分的块数
        num_row_splits = max(1, math.ceil(rows / max_rows))
        num_col_splits = max(1, math.ceil(cols / max_cols))
        
        for i in range(num_row_splits):
            for j in range(num_col_splits):
                r_start = i * max_rows
                c_start = j * max_cols
                r_end = min(r_start + max_rows, rows)
                c_end = min(c_start + max_cols, cols)
                
                section = WeightSection(
                    operator_id=operator.name,
                    rows=r_end - r_start,
                    cols=c_end - c_start,
                    depth=math.ceil((r_end - r_start) * (c_end - c_start) / (max_rows * max_cols))
                )
                sections.append(section)
        
        return sections
```

### 4.3 MoE 激活轨迹分析

#### 4.3.1 分析流程

```
1. 加载 activation_trace.json
2. 构建共现矩阵 C[N×N]
3. 计算每个 Expert 的激活频率
4. 识别互斥 Expert 对（C[i][j] == 0）
5. 按激活频率排序，生成优先级列表
```

#### 4.3.2 优化策略

- **高频 Expert**：优先放置在 z=0 位置（低切换开销）
- **共现 Expert**：尽量分配到不同 Sub-Cube（并行执行）
- **互斥 Expert**：可共享同一 Sub-Cube（节省空间）

#### 4.3.3 关键代码实现

```python
class TraceAnalyzer:
    """MoE激活轨迹分析器"""
    def analyze(self, trace_path: str) -> TraceAnalysis:
        with open(trace_path, 'r') as f:
            traces = json.load(f)
        
        num_experts = len(traces['experts'])
        cooccurrence = np.zeros((num_experts, num_experts))
        frequency = np.zeros(num_experts)
        
        for trace in traces['traces']:
            activated = trace['activated_experts']
            # 更新频率
            for expert_id in activated:
                frequency[expert_id] += 1
            # 更新共现矩阵
            for i in activated:
                for j in activated:
                    cooccurrence[i][j] += 1
        
        # 识别互斥对
        mutex_pairs = []
        for i in range(num_experts):
            for j in range(i+1, num_experts):
                if cooccurrence[i][j] == 0:
                    mutex_pairs.append((i, j))
        
        return TraceAnalysis(
            cooccurrence=cooccurrence,
            frequency=frequency,
            mutex_pairs=mutex_pairs,
            priority_order=np.argsort(-frequency)  # 频率降序
        )
```

### 4.4 3D 映射算法

#### 4.4.1 Greedy First-Fit 算法

```
for each section in priority_order:
    for each subcube in range(N*N):
        find first free z-position
        if fits:
            place section
            break
    if not placed:
        try least-loaded subcube
```

#### 4.4.2 优先级排序

1. 大权重优先（减少碎片）
2. MoE 高频 Expert 优先
3. 关键路径算子优先

#### 4.4.3 关键代码实现

```python
class Mapper3D:
    """3D映射器"""
    def map(self, sections: List[WeightSection], 
            config: HardwareConfig,
            trace_analysis: Optional[TraceAnalysis] = None) -> Solution:
        
        # 确定优先级顺序
        if trace_analysis:
            priority_order = self._get_priority_with_trace(sections, trace_analysis)
        else:
            priority_order = sorted(sections, key=lambda s: -s.rows * s.cols)
        
        placements = []
        subcube_usage = [0] * config.num_subcubes  # 跟踪每个Sub-Cube的使用情况
        
        for section in priority_order:
            placed = False
            # 尝试放置到每个Sub-Cube
            for sc_id in range(config.num_subcubes):
                z_pos = self._find_first_free_z(sc_id, section, config)
                if z_pos is not None:
                    placement = Placement3D(
                        section=section,
                        subcube_id=sc_id,
                        z_start=z_pos,
                        z_end=z_pos + section.depth
                    )
                    placements.append(placement)
                    subcube_usage[sc_id] += section.rows * section.cols
                    placed = True
                    break
            
            if not placed:
                # 放置到负载最轻的Sub-Cube
                sc_id = np.argmin(subcube_usage)
                z_pos = self._find_first_free_z(sc_id, section, config)
                if z_pos is not None:
                    placement = Placement3D(...)
                    placements.append(placement)
        
        return Solution(placements=placements)
```

### 4.5 流水线调度

#### 4.5.1 依赖图构建

```python
class DependencyGraph:
    nodes: Dict[str, Operator]
    edges: List[Tuple[str, str]]  # (predecessor, successor)
    
    def topological_sort(self) -> List[str]:
        """拓扑排序"""
        in_degree = {node: 0 for node in self.nodes}
        for _, succ in self.edges:
            in_degree[succ] += 1
        
        queue = [node for node, deg in in_degree.items() if deg == 0]
        result = []
        
        while queue:
            node = queue.pop(0)
            result.append(node)
            for _, succ in self.edges:
                if _ == node:
                    in_degree[succ] -= 1
                    if in_degree[succ] == 0:
                        queue.append(succ)
        
        return result
    
    def get_ready_nodes(self, completed: Set[str]) -> List[str]:
        """获取已就绪的节点（所有前驱已完成）"""
        ready = []
        for node in self.nodes:
            if node not in completed:
                predecessors = [pred for pred, _ in self.edges if _ == node]
                if all(pred in completed for pred in predecessors):
                    ready.append(node)
        return ready
```

#### 4.5.2 Barrier 同步

下游算子必须等待上游所有 Section 输出就绪：

$$T_{ready}(v) = \max_{u \in Predecessors(v)} T_{end}(u)$$

#### 4.5.3 关键代码实现

```python
class PipelineScheduler:
    """流水线调度器"""
    def schedule(self, solution: Solution, dep_graph: DependencyGraph) -> Schedule:
        topo_order = dep_graph.topological_sort()
        schedule = []
        completed = set()
        end_times = {}
        
        for op_id in topo_order:
            # 计算就绪时间（Barrier同步）
            predecessors = dep_graph.get_predecessors(op_id)
            ready_time = max([end_times.get(pred, 0) for pred in predecessors], default=0)
            
            # 计算执行时间
            placement = solution.get_placement(op_id)
            exec_time = self._calculate_exec_time(placement)
            
            start_time = ready_time
            end_time = start_time + exec_time
            
            schedule.append(ScheduledStep(
                operator_id=op_id,
                start_time=start_time,
                end_time=end_time,
                subcube_id=placement.subcube_id
            ))
            
            completed.add(op_id)
            end_times[op_id] = end_time
        
        return Schedule(steps=schedule, total_time=max(end_times.values()))
```

### 4.6 硬件模拟

#### 4.6.1 延迟计算

| 操作 | 延迟（周期） |
|------|-------------|
| Weight-Cube 激活 | 1 + D |
| Sub-Cube 内切换 | D |
| 跨 Sub-Cube 并行 | 0（完全并行） |

#### 4.6.2 约束验证

1. **空间互斥**：同一 Sub-Cube 内 Weight-Cube 不重叠
2. **深度约束**：z_end ≤ config.depth
3. **依赖正确性**：调度顺序满足拓扑序

#### 4.6.3 关键代码实现

```python
class HardwareSimulator:
    """硬件模拟器"""
    def validate(self, solution: Solution, config: HardwareConfig) -> ValidationResult:
        errors = []
        
        # 检查深度约束
        for placement in solution.placements:
            if placement.z_end > config.depth:
                errors.append(f"深度溢出: {placement.section.operator_id}")
        
        # 检查空间互斥
        for sc_id in range(config.num_subcubes):
            sc_placements = [p for p in solution.placements if p.subcube_id == sc_id]
            for i in range(len(sc_placements)):
                for j in range(i+1, len(sc_placements)):
                    if self._overlaps(sc_placements[i], sc_placements[j]):
                        errors.append(f"空间重叠: {sc_placements[i].section.operator_id} vs {sc_placements[j].section.operator_id}")
        
        return ValidationResult(is_valid=len(errors) == 0, errors=errors)
    
    def calculate_latency(self, schedule: Schedule, config: HardwareConfig) -> LatencyResult:
        total_cycles = schedule.total_time
        switch_overhead = self._calculate_switch_overhead(schedule, config)
        return LatencyResult(
            compute_cycles=total_cycles,
            switch_cycles=switch_overhead,
            total_cycles=total_cycles + switch_overhead
        )
```

## 5. 优化论证与分析

### 5.1 映射算法优化

#### 5.1.1 优化前：随机放置

- 空间碎片化严重
- 深度溢出风险高
- Sub-Cube 负载不均衡

#### 5.1.2 优化后：Greedy First-Fit + 优先级排序

- 空间紧凑，碎片最小化
- 自动适配深度需求
- 负载均衡，并行度最大化

### 5.2 MoE 优化策略

#### 5.2.1 共现感知分组

利用激活轨迹构建共现矩阵，将频繁共现的 Expert 分配到不同 Sub-Cube，最大化并行度。

#### 5.2.2 频率感知放置

高频 Expert 放置在 z=0 附近，降低切换开销。

### 5.3 调度优化

#### 5.3.1 关键路径优先

识别计算图的关键路径，优先调度关键路径上的算子。

#### 5.3.2 流水线并行

利用 Sub-Cube 间并行性，实现多层流水线并行执行。
