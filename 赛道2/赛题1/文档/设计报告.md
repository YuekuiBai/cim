# 赛道二：编译工具链 - 赛题一 设计报告

## 面向 NAND FLASH 存算一体硬件的编译架构设计

---

## 摘要

本报告针对基于 NAND FLASH 的存算一体（CIM）硬件，设计并实现了一套完整的编译框架。该框架能够将 ONNX 神经网络模型高效编译为目标硬件的 ISA 指令序列，涵盖 IR 设计、模型解析、资源管理、指令生成等核心模块。实验结果表明，该框架能够成功编译包含 linear 和 elementwise 算子的多种 ONNX 模型，并生成正确的硬件指令序列。

---

## 1. 引言

### 1.1 研究背景与意义

随着大模型技术的快速演进与边缘计算的普及，传统冯·诺依曼架构正面临"内存墙"与"能耗墙"的双重挑战。存算一体架构以其高能效比与高计算密度的优势，成为突破计算瓶颈的关键技术。

以 NAND FLASH 为代表的新型非易失性存储介质因其高密度、低功耗、低成本的特性，正被广泛应用于存算一体芯片的设计中。然而，如何将复杂的神经网络高效部署到基于 NAND FLASH 的存算一体硬件上，仍是一个涉及编译器、架构与算法协同优化的系统工程难题。

### 1.2 研究目标

本赛题旨在设计并实现一套面向存算阵列的编译器中间表示与编译框架，完成从神经网络模型到存算指令的高效映射与调度，推动存算一体技术在大模型边缘部署中的落地应用。

## 2. 工作原理与关键技术原理分析

### 2.1 基本概念

#### 2.1.1 存内计算（Computing-in-Memory, CIM）

存内计算技术直接在数据存储的区域进行计算，而非传统计算结构下频繁在存储单元和计算单元间进行数据传递。存内计算技术尝试解决冯·诺伊曼结构的内存速率瓶颈问题，降低通讯开销，提升特定使用场景下的计算效率。

#### 2.1.2 Bit-Serial 计算

本设计采用 bit-serial 计算方案，将输入数据按位拆分，逐位进行存内计算，最终通过移位累加得到完整结果。该方案的优势在于：

- 存算单元只需存储权重，输入数据按位串行输入
- 输出累加器使用 i32 精度，避免中间结果溢出
- 充分利用 CIM 阵列的并行性

### 2.2 处理流程

```
ONNX 模型 → 前端解析 → IR 构建 → 图优化 → SRAM 分配 → 权重映射 → 指令生成 → ISA 输出
```

### 2.3 数学建模

#### 2.3.1 Linear 算子的 Bit-Serial 分解

对于 Linear 算子 $Y = X \times W$，其中 $X \in \mathbb{Z}^{B \times C_{in}}$，$W \in \mathbb{Z}^{C_{in} \times C_{out}}$。

将输入 $X$ 按位分解：

$$X = \sum_{i=0}^{7} X^{(i)} \cdot 2^i$$

其中 $X^{(i)}$ 表示 $X$ 的第 $i$ 个 bit 平面。

则输出可表示为：

$$Y = \sum_{i=0}^{7} (X^{(i)} \times W) \cdot 2^i$$

对于 int8 有符号数，第 7 位（符号位）需要特殊处理：

$$Y = \sum_{i=0}^{6} (X^{(i)} \times W) \cdot 2^i - (X^{(7)} \times W) \cdot 2^7$$

### 2.4 硬件资源模型

| 资源 | 规格 | 约束 |
|------|------|------|
| 2D 存算单元 | 1024bit(row) × 4096bit(col) | 权重映射空间 |
| SRAM | 512KB | 输入/输出/中间数据存储 |

## 4. 详细设计与实现

### 4.1 IR 设计

#### 4.1.1 设计原则

- **表达能力**：支持 linear 和 elementwise（add/sub/mul/div）算子，能够完整表达赛题要求的所有计算图结构
- **简洁性**：最小化节点类型，仅定义 Linear 和 Elementwise 两种核心节点，统一数据流表示
- **扩展性**：预留 attributes 字段，支持未来扩展 Conv2d、BatchNorm、Activation 等算子

#### 4.1.2 IR 数据结构

```
IRGraph
├── nodes: List[IRNode]        # 计算节点列表
├── tensors: Dict[str, Tensor] # 张量信息（形状、数据类型）
└── edges: List[Edge]          # 数据依赖边（producer → consumer）
```

#### 4.1.3 节点类型

| 节点类型 | 说明 | 属性 |
|---------|------|------|
| Linear | 矩阵乘法 + 可选 bias | weight (int8), bias (int32) |
| Elementwise | 逐元素运算 | op_type (add/sub/mul/div) |

#### 4.1.4 关键代码实现

```python
class IRNode:
    """IR计算节点"""
    def __init__(self, node_id: str, op_type: str):
        self.id = node_id
        self.op_type = op_type  # "linear" or "elementwise"
        self.inputs: List[str] = []
        self.outputs: List[str] = []
        self.weight: Optional[np.ndarray] = None
        self.bias: Optional[np.ndarray] = None
        self.attributes: Dict[str, Any] = {}
        # 资源分配结果
        self.sram_input_addr: Optional[int] = None
        self.sram_output_addr: Optional[int] = None
        self.weight_mapping: Optional[WeightMapping] = None
```

### 4.2 模型解析

#### 4.2.1 ONNX 算子映射

| ONNX 算子 | IR 节点 | 说明 |
|-----------|---------|------|
| MatMul | Linear | 无 bias 的矩阵乘法 |
| Gemm | Linear | 支持 alpha/beta/transA/transB 属性 |
| Add | Elementwise | 逐元素加法 |
| Sub | Elementwise | 逐元素减法 |
| Mul | Elementwise | 逐元素乘法 |
| Div | Elementwise | 逐元素除法 |

#### 4.2.2 解析流程

```
1. 加载 ONNX 模型文件 (onnx.load)
2. 遍历计算图节点 (graph.node)
3. 根据 op_type 创建对应 IR 节点
4. 从 initializer 提取权重和 bias 数据
5. 构建张量依赖关系 (producer/consumer)
6. 拓扑排序确定执行顺序
```

#### 4.2.3 关键代码实现

```python
def convert_onnx_to_ir(onnx_model: onnx.ModelProto) -> IRGraph:
    """将ONNX模型转换为IR表示"""
    ir_graph = IRGraph()
    
    # 提取所有 initializer（权重常量）
    initializers = {init.name: numpy_helper.to_array(init) 
                    for init in onnx_model.graph.initializer}
    
    # 遍历计算图节点
    for node in onnx_model.graph.node:
        if node.op_type in ('MatMul', 'Gemm'):
            ir_node = _convert_linear(node, initializers)
        elif node.op_type in ('Add', 'Sub', 'Mul', 'Div'):
            ir_node = _convert_elementwise(node)
        ir_graph.add_node(ir_node)
    
    # 拓扑排序
    ir_graph.topological_sort()
    return ir_graph
```

### 4.3 图优化

#### 4.3.1 常量折叠

识别并预先计算图中可静态确定的常量表达式，减少运行时计算量。

**优化规则**：
- 如果节点的所有输入都是常量，则在编译时直接计算结果
- 将计算结果作为新的常量节点插入图中

#### 4.3.2 算子融合

将连续的 elementwise 操作融合为单个节点，减少 SRAM 读写开销。

**融合规则**：
- Add → Add 融合为单个 Add
- 连续的同类型 elementwise 可合并

### 4.4 SRAM 资源管理

#### 4.4.1 地址空间规划

| 区域 | 用途 | 分配策略 |
|------|------|----------|
| input | 算子输入数据 | 按拓扑序复用 |
| output | 算子输出数据 | 按拓扑序分配 |
| acc | 累加器（bit-serial 中间结果） | 固定分配 |
| tmp | 临时缓冲区（elt 计算） | 按需分配 |

#### 4.4.2 分配算法

采用拓扑序贪心分配策略：

1. 按拓扑序遍历计算节点
2. 为每个节点的输入/输出分配 SRAM 地址
3. 当输入数据不再被后续节点使用时，释放其地址空间
4. 确保总使用量不超过 512KB

#### 4.4.3 关键代码实现

```python
class SRAMAllocator:
    """512KB SRAM地址分配器"""
    CAPACITY = 512 * 1024  # 512KB
    
    def allocate(self, ir_graph: IRGraph) -> SRAMLayout:
        layout = SRAMLayout()
        live_ranges = {}  # tensor_name -> (start, end)
        next_addr = 0
        
        for node in ir_graph.topo_order:
            # 释放不再使用的输入
            for input_name in node.inputs:
                if not self._is_live(input_name, ir_graph, node):
                    if input_name in live_ranges:
                        layout.free(live_ranges[input_name])
                        del live_ranges[input_name]
            
            # 分配输出地址
            output_size = self._get_tensor_size(node.outputs[0])
            output_addr = layout.allocate(output_size)
            live_ranges[node.outputs[0]] = (output_addr, output_addr + output_size)
        
        return layout
```

### 4.5 权重映射

#### 4.5.1 映射规则

| 维度 | 映射方向 | 说明 |
|------|---------|------|
| $C_{in}$ | row 方向 | 输入特征维度 |
| $C_{out} \times$ 位宽 | col 方向 | 输出特征维度 |

#### 4.5.2 切分策略

当权重矩阵超出 CIM 阵列容量时：

- **行切分**：沿 $C_{in}$ 方向切分，分别计算后累加
- **列切分**：沿 $C_{out}$ 方向切分，并行计算

#### 4.5.3 映射计算

$$num\_rows = \lceil \frac{C_{in}}{1024} \rceil$$
$$num\_cols = \lceil \frac{C_{out} \times 8}{4096} \rceil$$

### 4.6 指令生成

#### 4.6.1 Bit-Serial 计算原理

Bit-Serial 计算的核心思想是将 int8 输入按位拆分，逐位进行存内计算，最终通过移位累加得到完整结果。

对于 int8 有符号数，数值表示为：
$$x = -x_7 \cdot 2^7 + \sum_{i=0}^{6} x_i \cdot 2^i$$

因此输出计算为：
$$Y = \sum_{i=0}^{6} (X^{(i)} \times W) \cdot 2^i - (X^{(7)} \times W) \cdot 2^7$$

#### 4.6.2 指令序列生成

对于每个 Linear 节点，生成以下指令序列：

```
// Bit 0: 直接累加
cim.bit.i8 acc, input, 0, weight_pos

// Bit 1-6: 左移后累加
for bit_index in 1..6:
    cim.bit.i8 tmp, input, bit_index, weight_pos
    elt.mul.i32.vi shifted, tmp, 2^bit_index, len
    elt.add.i32.vv acc, acc, shifted, len

// Bit 7 (符号位): 左移后减法
cim.bit.i8 tmp, input, 7, weight_pos
elt.mul.i32.vi shifted, tmp, 128, len
elt.sub.i32.vv acc, acc, shifted, len

// 最终结果拷贝
mem.copy.i32.i32 output, acc, len
```

#### 4.6.3 Elementwise 指令生成

```python
def generate_elt_instruction(op: str, dst: int, src1: int, src2: int, len: int) -> str:
    """生成elementwise指令"""
    mode = "vv" if isinstance(src2, int) else "vi"
    return f"elt.{op}.i32.{mode} /*dst*/{dst}, /*src1*/{src1}, /*src2*/{src2}, /*len*/{len}"
```

## 6. 总结与展望

### 6.1 工作总结

本设计实现了一套完整的面向 NAND FLASH 存算一体硬件的编译框架，主要贡献包括：

1. **IR 设计**：定义了简洁可扩展的中间表示，支持 linear 和 elementwise 算子
2. **模型解析**：实现了 ONNX 到 IR 的完整转换，支持 MatMul/Gemm 等算子
3. **资源管理**：设计了 512KB SRAM 的拓扑序分配算法和权重映射策略
4. **指令生成**：实现了 bit-serial 计算方案的指令降级，生成正确的 ISA 序列

### 6.2 技术亮点

- **Bit-Serial 计算**：充分利用 CIM 阵列特性，实现高效的 int8 矩阵乘法
- **拓扑序 SRAM 分配**：保证数据依赖正确性的同时最大化空间利用率
- **模块化架构**：各模块接口清晰，便于扩展和维护

### 6.3 未来展望

1. **支持更多算子**：扩展至 Conv2d、BatchNorm、Activation 等算子
2. **高级优化**：引入算子调度、循环展开、数据复用等优化策略
3. **MLIR 集成**：基于 MLIR 框架实现，利用其成熟的优化基础设施
4. **硬件仿真**：开发指令级仿真器，验证生成指令的正确性
