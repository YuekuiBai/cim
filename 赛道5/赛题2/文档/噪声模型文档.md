## 1. 概述

### 1.1 背景

存算一体（In-Memory Computing, IMC）芯片利用新型非易失性存储器（RRAM、PCM、MRAM等）的物理特性，在存储阵列中直接完成矩阵向量乘法运算。这种模拟计算方式虽然能实现极高的能效，但也引入了多种类型的噪声，显著影响计算精度。

### 1.2 噪声建模目标

| 目标 | 说明 |
| --- | --- |
| **准确性** | 准确反映真实硬件的噪声特性 |
| **完整性** | 覆盖所有主要噪声源 |
| **可配置性** | 支持不同噪声强度和类型的配置 |
| **计算效率** | 噪声注入不应显著增加训练时间 |

### 1.3 噪声模型来源

本噪声模型参考主办方提供的存算一体芯片噪声模型设计：
- **来源**: https://resources.modelscope.cn/third-part/files/sample_noise.py
- **论文参考**: CoRR 2025论文 "Extending Straight-Through Estimation for Robust Neural Networks on Analog CIM Hardware"

## 2. 噪声类型与来源

### 2.1 三大噪声类型

| 噪声类型 | 主要来源 | 统计特性 |
| --- | --- | --- |
| **编程误差** | 权重写入时的精度限制 | 加性高斯噪声，与权重幅值相关 |
| **非线性噪声** | 器件电导-电压特性非线性 | 乘性噪声，在饱和区域显著 |
| **输出误差** | ADC量化、热噪声、1/f噪声 | 量化噪声+加性高斯噪声 |

### 2.2 编程误差（Programming Noise）

**产生机制：**
权重编程过程中，由于编程脉冲的控制精度限制和器件的随机性，实际编程的电导值与目标值存在偏差。

**特性：**
- 与权重幅值相关的加性噪声
- 相邻器件间可能存在空间相关性
- 服从高斯分布：$\epsilon_{prog} \sim \mathcal{N}(0, \sigma_{prog}^2)$

### 2.3 非线性噪声（Nonlinear Noise）

**产生机制：**
新型存储器件的电导-电压特性呈现非线性行为，特别是在高电导（饱和）区域，非线性效应更加显著。

**特性：**
- 与输入幅度相关的乘性噪声
- 在饱和区域表现尤为明显
- 可建模为：$\epsilon_{nonlin} = \alpha \cdot \max(0, V - V_{th})$ 或类似形式

### 2.4 输出误差（Output Noise）

**产生机制：**
- ADC量化噪声：有限位宽量化带来的误差
- 热噪声（Johnson-Nyquist noise）：导体中电子热运动产生
- 1/f噪声：低频闪烁噪声
- 运算放大器失调电压

**特性：**
- 加性高斯噪声
- 与信号幅度无关
- 服从高斯分布：$\epsilon_{out} \sim \mathcal{N}(0, \sigma_{out}^2)$

### 2.5 串扰噪声（Crosstalk）

**产生机制：**
相邻存储单元之间通过地线、电源线或寄生电容产生的干扰。

**特性：**
- 空间相关性：与相邻单元状态相关
- 通常作为输出噪声的附加项处理

## 3. 数学模型

### 3.1 完整噪声模型

对于存算阵列中的矩阵向量乘法 $Y = WX$，实际输出为：

$$\tilde{Y} = (W + \Delta W_{prog} + \Delta W_{drift}) \odot (X + \Delta X_{nonlin}) + \Delta Y_{out} + \Delta Y_{crosstalk}$$

其中：
- $\Delta W_{prog}$: 编程误差
- $\Delta W_{drift}$: 权重漂移
- $\Delta X_{nonlin}$: 非线性误差
- $\Delta Y_{out}$: 输出噪声
- $\Delta Y_{crosstalk}$: 串扰噪声
- $\odot$: 元素级乘法

### 3.2 各分量详细模型

#### 3.2.1 编程误差

$$\Delta W_{prog} = W \cdot \epsilon_{prog}, \quad \epsilon_{prog} \sim \mathcal{N}(0, \sigma_{prog}^2)$$

其中 $\sigma_{prog}$ 是编程噪声标准差。

#### 3.2.2 权重漂移

$$\Delta W_{drift} = W \cdot \delta \cdot t$$

其中：
- $\delta$ 是漂移因子
- $t$ 是时间或操作次数

简化模型中，漂移可建模为高斯噪声：
$$\Delta W_{drift} = W \cdot \epsilon_{drift}, \quad \epsilon_{drift} \sim \mathcal{N}(0, \sigma_{drift}^2)$$

#### 3.2.3 非线性饱和

器件电导-电压特性建模为：

$$g(V) = \frac{g_{max}}{1 + \alpha e^{-\beta V}}$$

其中 $g_{max}$ 是最大电导，$\alpha$ 和 $\beta$ 是非线性参数。

简化模型中，非线性效应建模为：

$$y_{sat} = \begin{cases}
y + \alpha \cdot y^2 & \text{if } y > 0 \\
y - \beta \cdot y^2 & \text{if } y < 0
\end{cases}$$

#### 3.2.4 输出噪声

$$\Delta Y_{out} \sim \mathcal{N}(0, \sigma_{out}^2)$$

#### 3.2.5 串扰

$$\Delta Y_{crosstalk} = \gamma \cdot \sum_{j \in \text{neighbors}} W_j \cdot X_j$$

其中 $\gamma$ 是串扰因子。

### 3.3 简化综合模型

在实践中，我们使用简化的综合模型：

```python
def compute_noisy_output(W, X, noise_config, noise_strength=1.0):
    # 1. 编程误差
    prog_noise = noise_config['prog_noise_std'] * noise_strength * torch.randn_like(W)
    W_noisy = W + W * prog_noise

    # 2. 非线性饱和
    alpha = noise_config['nonlinear_alpha'] * noise_strength
    beta = noise_config['nonlinear_beta'] * noise_strength
    X_pos = F.relu(X)
    X_neg = F.relu(-X)
    X_noisy = X + alpha * X_pos ** 2 - beta * X_neg ** 2

    # 3. 矩阵向量乘法
    Y = F.linear(X_noisy, W_noisy)

    # 4. 输出噪声
    output_noise = noise_config['output_noise_std'] * noise_strength * torch.randn_like(Y)
    Y = Y + output_noise

    # 5. 串扰
    crosstalk = noise_config['crosstalk_factor'] * noise_strength * torch.mean(Y)
    Y = Y + crosstalk

    return Y
```

## 4. 参数设置

### 4.1 默认参数

| 参数 | 符号 | 默认值 | 说明 |
| --- | --- | --- | --- |
| prog_noise_std | $\sigma_{prog}$ | 0.01 | 编程噪声标准差 |
| drift_factor | $\sigma_{drift}$ | 0.005 | 权重漂移因子 |
| nonlinear_alpha | $\alpha$ | 0.1 | 正向饱和参数 |
| nonlinear_beta | $\beta$ | 0.05 | 负向饱和参数 |
| output_noise_std | $\sigma_{out}$ | 0.01 | 输出噪声标准差 |
| crosstalk_factor | $\gamma$ | 0.002 | 串扰因子 |

### 4.2 噪声强度缩放

噪声强度可以通过 `noise_strength` 参数进行全局缩放：

```python
noise_config = {
    'prog_noise_std': 0.01,
    'drift_factor': 0.005,
    'nonlinear_alpha': 0.1,
    'nonlinear_beta': 0.05,
    'output_noise_std': 0.01,
    'crosstalk_factor': 0.002
}

# 噪声强度 = 1.0 (标准)
noisy_output = inject_noise(output, noise_config, noise_strength=1.0)

# 噪声强度 = 0.5 (低噪声)
noisy_output = inject_noise(output, noise_config, noise_strength=0.5)

# 噪声强度 = 2.0 (高噪声)
noisy_output = inject_noise(output, noise_config, noise_strength=2.0)
```

### 4.3 不同噪声场景配置

| 场景 | prog_noise_std | drift_factor | nonlinear_alpha | nonlinear_beta | output_noise_std |
| --- | --- | --- | --- | --- | --- |
| **低噪声** | 0.005 | 0.002 | 0.05 | 0.025 | 0.005 |
| **标准噪声** | 0.01 | 0.005 | 0.1 | 0.05 | 0.01 |
| **高噪声** | 0.02 | 0.01 | 0.2 | 0.1 | 0.02 |
| **极端噪声** | 0.05 | 0.025 | 0.5 | 0.25 | 0.05 |

### 4.4 参数选择依据

参数选择基于以下考虑：

1. **prog_noise_std**: 典型PCM/RRAM器件的编程精度约为1-2% [1]
2. **drift_factor**: 权重漂移通常为编程值的0.5-1% [2]
3. **nonlinear_alpha/beta**: 基于器件电导-电压特性实测数据
4. **output_noise_std**: 8-10位ADC的量化误差约为0.5-1%
5. **crosstalk_factor**: 典型值在0.1-0.3% [3]

| **噪声缩放测试** | 验证噪声强度参数的有效性 |
| **与参考实现对比** | 与主办方提供的噪声模型对比 |

### 5.2 统计特性验证

```python
def verify_noise_statistics():
    noise_config = get_default_noise_config()
    injector = NoiseInjector(noise_config)

    # 收集大量噪声样本
    samples = []
    for _ in range(10000):
        x = torch.randn(128, 512)
        noisy_x = injector.inject_to_output(x)
        noise = noisy_x - x
        samples.append(noise)

    samples = torch.cat(samples)

    # 验证统计特性
    mean = samples.mean().item()
    std = samples.std().item()

    print(f"Mean: {mean:.6f} (expected: ~0)")
    print(f"Std: {std:.6f} (expected: ~{noise_config['output_noise_std']})")

    # Shapiro-Wilk正态性检验
    from scipy import stats
    stat, p_value = stats.shapiro(samples[:1000].numpy())
    print(f"Shapiro-Wilk test: p={p_value:.4f}")
```

## 附录：完整噪声注入代码

```python
class NoiseInjector:
    def __init__(self, config: Dict[str, float]):
        self.config = config
        self.prog_noise_std = config.get('prog_noise_std', 0.01)
        self.drift_factor = config.get('drift_factor', 0.005)
        self.nonlinear_alpha = config.get('nonlinear_alpha', 0.1)
        self.nonlinear_beta = config.get('nonlinear_beta', 0.05)
        self.output_noise_std = config.get('output_noise_std', 0.01)
        self.crosstalk_factor = config.get('crosstalk_factor', 0.002)

    def inject_programming_noise(self, weight: torch.Tensor) -> torch.Tensor:
        noise = torch.randn_like(weight) * self.prog_noise_std
        return weight + weight * noise

    def inject_drift(self, weight: torch.Tensor) -> torch.Tensor:
        drift = torch.randn_like(weight) * self.drift_factor
        return weight + weight * drift

    def inject_nonlinearity(self, x: torch.Tensor) -> torch.Tensor:
        x_pos = F.relu(x)
        x_neg = F.relu(-x)
        return x + self.nonlinear_alpha * x_pos ** 2 - self.nonlinear_beta * x_neg ** 2

    def inject_output_noise(self, y: torch.Tensor) -> torch.Tensor:
        noise = torch.randn_like(y) * self.output_noise_std
        return y + noise

    def inject_crosstalk(self, y: torch.Tensor) -> torch.Tensor:
        crosstalk = self.crosstalk_factor * y.mean()
        return y + crosstalk

    def inject_to_output(self, output: torch.Tensor) -> torch.Tensor:
        y = self.inject_output_noise(output)
        y = self.inject_crosstalk(y)
        return y
```
